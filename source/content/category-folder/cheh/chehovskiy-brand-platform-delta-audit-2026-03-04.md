## 4. Color-delta анализ (расширенный)

### 4.1 Что зафиксировано в бренд-платформе (факты)

Из части 1 (`Визуальные ориентиры`):
1. Цветовой каркас: бордовый (основа), зеленый (ПП), нейтрали (фон).
2. ПП-коммуникация прямо завязана на зеленую маркировку.

Из части 3 (`Цвет и композиция`, `Упаковка и POSM`, `Фото/видео-гайд`):
1. Бордовый задан как главный брендовый слой.
2. Зеленый ограничен функциональной ролью ПП.
3. Нейтрали вынесены в служебный и фоновой слой.
4. Даны запреты: неон, розовый, конфликтные контрастные пары.
5. Есть категория-логика:
   - горячее: нейтраль + бордовый;
   - ПП: светлые решения + зеленые маркеры;
   - sushi: нейтраль + бордовые акценты;
   - десерты: бордовый + теплые акценты.

### 4.2 Что отсутствует в финальном ТЗ (конкретно)

В `catering-landing-tz-final.md` нет color-контракта уровня реализации:
1. Нет токенов/переменных (`--color-*`) для дизайна и фронта.
2. Нет таблицы `роль цвета -> UI блок -> допустимое применение`.
3. Нет color-правил для веток `family/office/executive`.
4. Нет связки `ad_hook -> цветовой акцент` для рекламных креативов.
5. Нет проверяемых критериев контрастности для текста/CTA/proof-плашек.

### 4.3 Что не договорено даже в бренд-доках (важно)

1. Нет числовой палитры (HEX/RGB/CMYK/LAB), только семантика.
2. Нет шкал оттенков (50/100/200...) для состояния UI (hover/disabled/background).
3. Нет правила конвертации печать -> digital для сохранения единого оттенка бордового.

Это не ошибка бренд-платформы как стратегии, но это блокер для стабильной реализации в продукте и рекламе.

### 4.4 Почему это влияет на конверсию, а не только на «красоту»

1. Разные команды начнут по-разному трактовать бордовый/зеленый, и креатив теряет узнаваемость в верхе воронки.
2. ПП-сигнал размывается, если зеленый уходит в нецелевые CTA/баннеры.
3. Без контрастных правил часть CTA/цены станет слабее на мобильных экранах, что бьет по `calc_start` и `card_expand`.

### 4.5 Что обязательно добавить в ТЗ (минимальный исполняемый набор)

1. Раздел `Brand Design Specs / Color Contract`.
2. `Color Token Table`:
   - `primary_bordo`,
   - `pp_green`,
   - `neutral_bg`,
   - `neutral_text`,
   - `accent_warm`,
   - `error/warning/success`.
3. `Usage Matrix` по экранам:
   - hero,
   - калькулятор,
   - SKU-карточка,
   - proof-блок,
   - CTA,
   - ad creatives.
4. Аватарная карта:
   - family: теплый бордовый + мягкие нейтрали;
   - office: нейтральный функциональный контур + аккуратные акценты;
   - executive: бордовый + высокий контраст + строгая типографическая подача.
5. Ограничение: зеленый только для ПП/health-сигналов.
6. Приемка: контраст текста/кнопок и визуальная consistency между креативом и лендингом.

## 5. Что еще потерялось при переносе в ТЗ

### 5.1 Стратегия и язык

1. Миссия/видение не оформлены как обязательные требования к лендинговому копирайту.
2. Потеряна часть персонификации: 5 персон в платформе сжаты до 3 аватаров без карты соответствия.
3. Фразовая библиотека не разложена по блокам интерфейса.

### 5.2 GTM и операции

1. Промо-календарь из части 2 не включен как рабочий ритм контента/креативов.
2. Офлайн-практики (`Слово Чеховского`, `Штамп заботы`) отражены концептуально, но без KPI-операционки.

### 5.3 Внедрение и продакшн

1. Нет приложения с production-спеками (иконки, размеры, носители, экспорт).
2. Нет роли-владельца визуальной целостности и handoff-процесса в рамках ТЗ.
