## 6) Критичные интерпретации

1. Для ПК по `furshetmenu` подтвержден устойчивый провал сценария звонка: `0` phone goal на всей глубине истории.
2. Формы в `furshetmenu` есть, но слабые и нерегулярные (`17` за всю историю), checkout-старта нет (`0`).
3. Для бизнес-решений по «продает/не продает звонок» desktop-сценарий текущего раздела классифицируется как неэффективный.

## 7) Ограничения и правила чтения этих данных

- Фильтр `startURLPath` означает, что сессия **началась** с выбранного пути; часть целей может достигаться позже на других страницах той же сессии.
- Автоцель «клик по номеру телефона» зависит от корректной разметки кликабельного номера (`tel:` или корректно настроенного трекинга).
- Пользователи (`users`) не суммируются из дневных строк как уникальные за долгий период; для уникальных значений используйте агрегаты API напрямую.
