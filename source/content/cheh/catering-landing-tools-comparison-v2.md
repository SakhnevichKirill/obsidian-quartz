# Сравнительный анализ сервисов для задачи лэндинга и performance-маркетинга (catering, РФ)

Дата ревизии: 4 марта 2026

## 1) Что сравниваем
Сервисы: `SpyWords`, `Keys.so`, `Similarweb`, `SpyMetrics`, `Be1`.

Целевые задачи:
1. Отслеживать изменения спроса и предложения на рынке.
2. Понимать, куда таргетируются конкуренты и где есть шанс выиграть.
3. Контролировать, попадаем ли в топ выдачи.

## 2) Интерфейсы входа/выхода (критично для интеграции)
| Сервис | Входной интерфейс | Выходной интерфейс | Что важно для интеграции |
|---|---|---|---|
| SpyWords | `GET https://api.spywords.ru/?method=...` + `login/token`, `se=yandex/google`, `site/word`, `offset/rows`, `asc/desc` | Табличный ответ (TSV): 1-я строка `ERROR` или заголовки, далее строки данных | Легко вызвать, но нужен парсер TSV и обработка `ERROR xx`; формат не JSON |
| Keys.so | REST JSON на `https://api.keys.so`; auth: `X-Keyso-TOKEN` (header) или `auth-token` (query); mix `GET/POST/DELETE/PATCH`; есть async-паттерн `POST` + `.../state/<uid>` | JSON-объекты/списки, часто пагинация `current_page/per_page/last_page/total/data[]` | Самый удобный для продуктовой интеграции; можно строить пайплайны и мониторинг |
| Similarweb | REST API (`v1/v4`), обычно `api_key` в query; примеры endpoint-ов: website traffic, keyword search, rank-tracker | JSON (метрики, списки, таймсерии) | Интерфейс зрелый, но важно ограничение Search 3.0 по источнику поисковиков |
| SpyMetrics | REST `GET https://api.spymetrics.ru/v1/website/{domain}/...` + `api_key`, `start_date/end_date`, `granularity`, `country`, `main_domain_only`, `format=json/xml` | JSON или XML; типовой ответ: `meta` + `data[]`/`visits[]` | Простая модель для батч-аналитики; лимит 5 req/s на аккаунт |
| Be1 | Асинхронный API: `POST /api/tools/add-task` -> `slug` -> `GET /api/tools/get-result` | JSON: `status`, `result`, `position`, `message` | Дешево и удобно для пакетных SEO-задач, но есть очередь и beta-режим |

## 3) Что реально дают данные под ваши задачи
| Задача | SpyWords | Keys.so | Similarweb | SpyMetrics | Be1 |
|---|---|---|---|---|---|
| Динамика спроса | Частично (история по доменам/ключам, региональные базы) | Хорошо (wordstat + keyword/serp + истории) | Хорошо для макро, но Search 3.0 volume = Google-only | Хорошо по доменам/трафику/конкурентам | Локально хорошо (wordstat, кластеризация), макро — слабо |
| Куда таргетируются конкуренты | Сильно (DomainAdv, объявления, конкуренты в контексте) | Сильно (context/direct блоки, конкуренты, ads/keywords) | Средне (рынок и доли, не фокус на Yandex.Direct деталях) | Средне (paid competitors, traffic sources) | Слабо |
| Где выигрывать аукцион | Слабо/средне (оценочные CPC/конкурентность) | Средне (оценочные поля по контексту) | Слабо (не Yandex auction-native) | Слабо (оценочные агрегаты) | Слабо |
| Попадаем ли в топ поиска | Хорошо (organic позиции и сравнения) | Очень хорошо (monitoring, serp, rank/динамика) | Средне/хорошо (rank-tracking API как дополнительный контур) | Средне (keywords/competitors без полноценного мониторинга позиций как основной функции) | Ограниченно |

## 4) Критический разбор по каждому сервису

### SpyWords
Сильные стороны:
1. Самый релевантный для RU-кейсов по Яндекс/Google в одной модели `method`.
2. Есть конкурентные методы именно по контексту и объявлениям: `DomainAdv`, `DomainAdvCompetitors`, `FightAdv`.
3. Есть методы по органике и топам: `DomainOrganic`, `CompareDomainTop10`.

Риски/ограничения:
1. Выход не JSON, а TSV; выше риск ошибок ETL и типизации.
2. Legacy auth (`login/token`) и legacy-style API.
3. Метрики по ставкам/бюджетам — оценочные, не «истина аукциона».

Вердикт: сильный источник конкурентной разведки в Яндексе, но технически менее удобен, чем modern REST JSON API.

### Keys.so
Сильные стороны:
1. Самый зрелый интерфейс для автоматизации: много endpoint-ов, пагинация, async state.
2. Есть важные для вас контуры: `direct`, `organic`, `serp`, `monitoring`, `wordstat`.
3. Региональная модель `base` с Яндекс-базами (msk/spb/ekb и др.) и Google-базами.

Риски/ограничения:
1. Часть задач асинхронная, нужна дисциплина оркестрации (polling/retry/idempotency).
2. Значительная часть показателей в рекламе и трафике оценочная.
3. По сложным сценариям нужно жестко валидировать качество на ваших тестовых запросах.

Вердикт: лучший кандидат на «ядро» интеграции и регулярной аналитики.

### Similarweb
Сильные стороны:
1. Корпоративный стандарт API, хорошие макро-метрики и кросс-рынок.
2. Есть выделенные продукты `Website Keywords Search 3.0`, `Rank Tracking API`.

Риски/ограничения:
1. По официальной поддержке Search 3.0: search volume возвращается только для Google.
2. Для РФ/Яндекс-центричной ниши ценность ниже, чем у RU-специализированных сервисов.
3. Для задач аукциона Яндекс.Директа сервис не является нативным источником.

Вердикт: полезен как макро-слой и бенчмарк, но не как основной источник в вашей конфигурации.

### SpyMetrics
Сильные стороны:
1. Простой REST и понятные таймсерии по трафику/источникам/конкурентам.
2. Есть RU country-filter и paid/organic competitor-блоки.
3. Удобен для «панорамного» мониторинга доменов.

Риски/ограничения:
1. Не заточен на «аукционную правду» по Яндекс.Директ.
2. Есть ограничения по покрытию/доступности отдельных keyword-наборов.
3. Лимит запросов на аккаунт 5 req/s.

Вердикт: сильный дополнительный источник для трендов и сравнений, не ядро для аукционных решений.

### Be1
Сильные стороны:
1. Дешевый инструментальный API для потока SEO-операций.
2. Четкий async-контракт (`add-task` / `get-result`) и понятная стоимость по инструментам.
3. Практически полезен для wordstat, группировки и семантического расширения.

Риски/ограничения:
1. API в beta-режиме.
2. Не закрывает рыночную конкурентную разведку уровня Direct/auction.
3. Модель очереди добавляет задержку в production-циклы.

Вердикт: хороший low-cost модуль семантики, но не полноценный конкурентный intelligence.

## 5) Сводный приоритет под ваш кейс
Если цель — именно RU/Yandex performance для кейтеринга:
1. `Keys.so` — основной API-контур (операционный и аналитический).
2. `SpyWords` — второй контур для углубленной конкурентной разведки по Яндекс.Директ/органике.
3. `Be1` — дешевый семантический доп. слой.
4. `SpyMetrics` — макро-дополнение по трафику/конкурентам.
5. `Similarweb` — опционально, если нужен международный/кросс-рынок бенчмарк.

## 6) Важный вывод (без иллюзий)
Ни один из 5 сервисов не дает «чистую аукционную истину» уровня вашей учетной записи Яндекс.Директа (прогноз/фактическая аукционная конкуренция в вашем аккаунте по вашим кампаниям) в полном виде как в нативных инструментах Директа.

Практически это значит:
1. Эти сервисы нужны для разведки и гипотез.
2. Финальное решение по ставкам и аукциону нужно подтверждать данными из нативного контура Яндекс.Директа.
